# CargoDefect-YOLOv26-Detect — 货件缺陷检测系统使用教程

---

## 1. 项目简介

基于 YOLOv26 的实时货件缺陷检测 GUI。支持：

- 摄像头实时推理（后台线程，不卡界面）
- YOLO 包裹/缺陷检测 + ROI 裁剪
- 背景差分兜底（YOLO 漏检时）
- 20 张样本评分与报表

**包裹判定优先级：** YOLO → 背景差分 → 固定工位 → 无（待检）

## 3. 迁移到目标电脑（重要）

### 正确做法

1. 复制整个 `deploy_gui` 文件夹
2. **不要复制 `venv/`**（或复制后先删掉）
3. 路径尽量用英文，例如 `D:\deploy_gui`
4. 目标机安装 Python 3.10+，勾选 **Add Python to PATH**
5. 双击 **`run.bat`**，等待 3–5 分钟自动安装

### 错误做法

- 把源电脑的 `venv` 一起拷过去 → 路径写死、DLL 不匹配 → 卡顿或直接报错
- 项目放在含中文的路径下 → 可能编码异常

## 4. 快速启动

双击 `run.bat`，或：

```powershell
cd deploy_gui
python -m venv venv
venv\Scripts\activate
pip install -r requirements_gui.txt
pip install --force-reinstall --no-deps "ultralytics-main"
python app.py
```

## 5. 推荐操作流程

| 步骤 | 操作 |
|------|------|
| 1 | 加载模型 → 打开摄像头 |
| 2 | **清空工位** → 点「采集空工位背景」 |
| 3 | 放入纸箱，确认「包裹来源」不是「无」 |
| 4 | 「采集单张」可立即抓拍（不要求有包裹） |
| 5 | 「采集20张」需检测到包裹才会自动抓拍 |
| 6 | 对样本点正确/错误 → 保存报表 |

## 6. 画面卡顿 / 无法采集 — 解决方案

### 画面卡顿

| 原因 | 解决 |
|------|------|
| 推理堵界面 | 已改为后台推理线程，请更新最新代码后重试 |
| 复制了旧 venv | 删除目标机上的 `venv`，重新运行 `run.bat` |
| CPU 较慢 | 增大 `config.yaml` → `performance.frame_skip`（如 8→12） |
| 分辨率偏高 | `model.imgsz: 416`；摄像头改为 `640×360` |

### 无法采集

| 原因 | 解决 |
|------|------|
| 自动采集要求有包裹 | 一直「待检」时不会自动抓拍 |
| 未采空背景 | 先采空背景，再放纸箱 |
| 想强制采集 | 用「采集单张」（即使待检也可抓） |
| 临时兜底 | 勾选「固定工位模式」 |

## 7. 关键配置（config.yaml）

```yaml
model:
  weights: weights/best_cargodefect_yolov26_detect.pt
  imgsz: 640          # 卡顿可改 416
  device: cpu         # 有 GPU 可改 0

threshold:
  package_conf: 0.03
  defect_show_conf: 0.12
  defect_ng_conf: 0.25

performance:
  frame_skip: 8       # 越大越流畅，越小越实时

package_fallback:
  enable: true
  min_diff_ratio: 0.04

camera:
  source: 0           # 打不开就试 1、2
```

## 8. 故障排查

| 问题 | 解决 |
|------|------|
| run.bat 闪退 | 安装 Python 并勾选 PATH |
| 模型加载失败 | `pip install --force-reinstall --no-deps ultralytics-main` |
| 摄像头打不开 | 改 `camera.source` |
| 一直待检 | 采空背景 / 固定工位 / 降低 package_conf |
| 路径乱码 | 放到纯英文路径 |
